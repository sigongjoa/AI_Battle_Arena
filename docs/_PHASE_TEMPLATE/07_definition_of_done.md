## 📋 개요

**Definition of Done (DoD)**은 개발 작업이 완료되었다고 판단하기 위한 명확한 기준입니다.

이 문서는 Phase X에서 모든 작업 항목이 만족해야 하는 기준을 정의합니다.

---

## 🎯 DoD의 목적

- ✅ 완료 기준의 명확화
- ✅ 품질 일관성 유지
- ✅ 재작업 감소
- ✅ 팀 간 기대치 동일화
- ✅ 릴리스 준비도 확인

## 🚀 DoD 엄격성 수준

### Level 1: 최소 (MVP)
- 코드 작성 완료
- 단위 테스트 통과
- 기본 문서화

**적합한 경우**: 프로토타입, 실험적 기능

### Level 2: 표준 (Production Ready)
- 모든 Level 1 기준 충족
- 통합 테스트 통과
- 코드 리뷰 완료
- 보안 검토 완료
- 성능 테스트 통과

**적합한 경우**: 대부분의 기능 구현

### Level 3: 엄격 (Enterprise)
- 모든 Level 2 기준 충족
- E2E 테스트 통과
- 사용자 테스트 완료
- 모니터링 설정
- SLA 기준 충족
- 24/7 지원 준비

**적합한 경우**: 미션 크리티컬 기능

## 🔄 DoD 변경 관리

### DoD 추가 요청 프로세스

1. **제안**: 팀원이 필요한 기준 제안
2. **검토**: Phase 리드 및 아키텍트 검토
3. **토론**: 팀 회의에서 영향도 논의
4. **승인**: 리드의 최종 승인
5. **적용**: 다음 작업부터 적용

### DoD 제거 요청

예외적 상황 (성능 이슈, 마감 촉박):
- 리드 + 스테이크홀더 합의 필수
- 이유 문서화
- 향후 보완 일정 수립

## 📈 DoD 효과 측정

### 추적 지표

```
1. 완료 작업 재작업율
   목표: 5% 이하
   측정: (재작업 작업수 / 완료 작업수) × 100

2. DoD 준수율
   목표: 95% 이상
   측정: (DoD 충족 작업 / 전체 작업) × 100

3. 버그 밀도
   목표: 0.5 bugs/KLOC 이하
   측정: 발견된 버그수 / 1000 라인 코드

4. 테스트 커버리지
   목표: 80% 이상
   측정: (테스트된 라인 / 전체 라인) × 100

5. 배포 전 버그 발견율
   목표: 95% (배포 후 발견 최소화)
   측정: (배포 전 발견 / 전체 발견) × 100
```

## ✅ Phase X DoD 최종 체크

**DoD 적용 대상**:
- [ ] 모든 기능 구현 작업
- [ ] 모든 버그 수정 작업
- [ ] 모든 리팩토링 작업

**DoD 검증 책임자**:
- 개발자: 자체 검증
- 코드 리뷰어: 코드 품질 검증
- QA: 기능 및 성능 검증
- 리드: 최종 승인

**DoD 위반 시 처리**:
- Minor (1-2개 미충족): 경고 + 보완 일정 수립
- Major (3개 이상 미충족): 작업 반려 + 재작업 필요
- Critical (보안/성능): 즉시 중단 + 긴급 조치
